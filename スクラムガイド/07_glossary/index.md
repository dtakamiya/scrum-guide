# アジャイル・スクラム用語集（現場実践・英日対訳）

## 概要

本用語集は、アジャイル・スクラム・多ベンダー協業・KPI・トラブルシューティングの現場で頻出する最新用語・略語・英語/日本語対訳・現場Tipsをまとめたものです。主要な海外・国内サイトの知見をもとに、実践的な解説を付記しています。

---

| 用語（英語） | 用語（日本語） | 解説・現場Tips |
|--------------|----------------|----------------|
| Agile | アジャイル | 変化に柔軟に対応し、顧客価値を最大化する開発手法・マインドセット。 |
| Scrum | スクラム | アジャイルの代表的フレームワーク。短いスプリントで反復開発。 |
| Sprint | スプリント | 1～4週間の開発サイクル。計画→実装→レビュー→ふりかえりを繰り返す。 |
| Product Owner (PO) | プロダクトオーナー | プロダクトの価値最大化責任者。優先順位決定・顧客調整。 |
| Proxy PO | プロキシPO | POの代理。大規模・多ベンダー体制でよく登場。 |
| Scrum Master | スクラムマスター | プロセス促進・障害除去・チーム支援役。心理的安全性の醸成も担う。 |
| Developer | 開発者 | 実装・テスト・設計などを担うチームメンバー。 |
| Stakeholder | ステークホルダー | 利害関係者。顧客・経営層・他部門など。 |
| Backlog | バックログ | やるべき作業・要件のリスト。優先順位付き。 |
| User Story | ユーザーストーリー | 顧客視点での要件記述。「誰が・何を・なぜ」形式。 |
| Definition of Done (DoD) | 完了の定義 | 「完了」とみなす基準。品質・テスト・レビュー等を明文化。 |
| Retrospective | ふりかえり | スプリントごとに実施する改善会議。KPTやSailboat等の手法あり。 |
| Impediment | インピーディメント | チームの進行を妨げる障害・課題。早期発見・可視化が重要。 |
| Kanban | カンバン | タスクの「見える化」ボード。WIP制限やフロー改善に活用。 |
| Velocity | ベロシティ | 1スプリントで完了したストーリーポイントの合計。計画精度向上に活用。 |
| Lead Time | リードタイム | 要件→リリースまでの所要時間。短縮が価値提供の鍵。 |
| Cycle Time | サイクルタイム | 作業開始→完了までの時間。ボトルネック発見に有効。 |
| WIP (Work in Progress) | 仕掛かり作業 | 同時進行中のタスク数。多すぎると効率低下。 |
| EBM (Evidence-Based Management) | 証拠ベース経営 | 指標・データに基づく意思決定。Scrum.org推奨。 |
| KPI (Key Performance Indicator) | 重要業績評価指標 | 成果・フロー・学習など多面的に設計。 |
| NPS (Net Promoter Score) | ネットプロモータースコア | 顧客満足度指標。「他者に薦めたいか」で測定。 |
| CI/CD (Continuous Integration/Continuous Delivery) | 継続的インテグレーション/デリバリー | 自動化による品質・リリース速度向上。 |
| RACI | RACIチャート | 責任分担表。Responsible, Accountable, Consulted, Informedの略。 |
| T&M (Time and Materials) | 時間・材料契約 | 作業時間・コストに応じて支払う契約形態。 |
| SOW (Statement of Work) | 作業範囲記述書 | 契約時に作業範囲・責任を明文化。 |
| Psychological Safety | 心理的安全性 | 「言いにくいこと」も言える雰囲気。チームの生産性向上に不可欠。 |
| Anti-pattern | アンチパターン | 失敗しやすい典型例。現場での注意喚起に活用。 |
| Cross-functional | クロスファンクショナル | 多様なスキルを持つメンバーで構成されたチーム。 |
| Self-organization | 自己組織化 | チームが自律的に意思決定・改善を行う文化。 |
| Burndown Chart | バーンダウンチャート | 残作業量の推移を可視化。進捗管理に活用。 |
| Burnup Chart | バーンアップチャート | 完了作業量の推移を可視化。全体像の把握に有効。 |
| Scrum of Scrums | スクラムオブスクラム | 複数チーム連携のための調整会議。 |
| PI Planning (Program Increment Planning) | PI計画 | SAFe等で使われる大規模計画会議。 |
| DoR (Definition of Ready) | 着手の定義 | 作業開始のための準備条件。 |
| Technical Debt | 技術的負債 | 将来の開発効率を下げる未解決の技術課題。 |
| Team Topologies | チームトポロジー | 組織設計のフレームワーク。ストリームアラインド/プラットフォーム等のチーム分類。 |
| Obeya | オベヤ | 大部屋。全体最適・情報共有のための物理/仮想スペース。 |
| Inspect and Adapt | 振り返りと適応 | SAFe等で使われる継続的改善の考え方。 |
| Servant Leadership | サーバントリーダーシップ | 支援型リーダーシップ。チームの成長・自律を促す。 |
| MVP (Minimum Viable Product) | 実用最小限プロダクト | 最小限の機能で市場検証する手法。 |
| OKR (Objectives and Key Results) | 目標と主要な成果 | 野心的な目標と測定可能な成果指標のセット。 |
| Mad-Sad-Glad | マッド・サッド・グラッド | ふりかえり手法の一つ。感情に着目。 |
| KPT (Keep, Problem, Try) | KPT | ふりかえり手法。「続けること・問題・試すこと」を整理。 |
| Sailboat | セイルボート | ふりかえり手法。推進力・障害・目標・リスクを可視化。 |
| Kanban Board | カンバンボード | タスクの進捗を「見える化」するボード。 |
| Standup Meeting | スタンドアップミーティング | 毎日の短い進捗共有会議。デイリースクラムとも。 |
| Escalation | エスカレーション | 権限を超える課題を上位者に報告・相談すること。 |
| Knowledge Sharing | ナレッジ共有 | 成功・失敗事例や学びをチーム・組織で共有。 |

---

> 本用語集は2025年7月時点の主要な国内外アジャイル・現場実践知をもとに作成しています。
